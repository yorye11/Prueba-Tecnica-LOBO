# Movie Finder - Prueba Técnica Fullstack

Aplicación web para buscar películas utilizando la API de OMDb y guardar favoritos por usuario. Desarrollada con React + Vite en el frontend y Node.js + Express en el backend, con autenticación y base de datos en Supabase.

## Tecnologías utilizadas

### Frontend

- React 18
- Vite
- Tailwind CSS
- Supabase Auth (autenticación)

### Backend

- Node.js
- Express
- PostgreSQL (Supabase)
- Axios

## Requisitos previos

- Node.js v18 o superior
- npm
- Cuenta en Supabase (https://supabase.com)
- API Key de OMDb (https://www.omdbapi.com/apikey.aspx)

## Instalación

### 1. Clonar el repositorio

```bash
git clone https://github.com/yorye11/Prueba-Tecnica-LOBO.git
cd Prueba-Tecnica-LOBO
```

### 2. Instalar dependencias del Backend

```bash
cd Backend
npm install
```

### 3. Instalar dependencias del Frontend

```bash
cd ../Frontend
npm install
```

## Variables de entorno

### Backend (Backend/.env)

Crear un archivo `.env` en la carpeta `Backend` con las siguientes variables:

```env

DATABASE_URL=postgresql://postgres:FullstackLOBO@db.zfdqhqabsugpljfvccpc.supabase.co:5432/postgres
OMDB_API_KEY=b807c40a
```

| Variable     | Descripción                                               |
| ------------ | --------------------------------------------------------- |
| PORT         | Puerto donde correrá el servidor (default: 3000)          |
| DATABASE_URL | URL de conexión a la base de datos PostgreSQL de Supabase |
| OMDB_API_KEY | API Key obtenida de OMDb para buscar películas            |

### Frontend (Frontend/.env)

Crear un archivo `.env` en la carpeta `Frontend` con las siguientes variables:

```env
VITE_SUPABASE_URL=https://zfdqhqabsugpljfvccpc.supabase.co

VITE_API_URL=https://prueba-tecnica-lobo.onrender.com
```

| Variable               | Descripción                          |
| ---------------------- | ------------------------------------ |
| VITE_SUPABASE_URL      | URL de el proyecto en Supabase       |
| VITE_SUPABASE_ANON_KEY | Clave pública (anon key) de Supabase |
| VITE_API_URL           | URL del backend desplegado           |

## Configuración de la base de datos

Ejecutar el siguiente script SQL en el editor de Supabase (SQL Editor):

```sql
CREATE TABLE favorites (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    imdb_id TEXT NOT NULL,
    user_id UUID NOT NULL,
    title TEXT NOT NULL,
    year TEXT,
    poster TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    CONSTRAINT unique_user_movie UNIQUE(user_id, imdb_id)
);
```

## Ejecución del proyecto

### 1. Iniciar el Backend

```bash
cd Backend
npm run dev
```

El servidor estará disponible en `http://localhost:3000`

### 2. Iniciar el Frontend

En una nueva terminal:

```bash
cd Frontend
npm run dev
```

La aplicación estará disponible en `http://localhost:5173`

## Endpoints de la API

| Método | Endpoint                 | Descripción                     |
| ------ | ------------------------ | ------------------------------- |
| GET    | /api/movies?s={query}    | Buscar películas por título     |
| GET    | /api/favorites/{user_id} | Obtener favoritos de un usuario |
| POST   | /api/favorites           | Agregar película a favoritos    |
| DELETE | /api/favorites/{id}      | Eliminar película de favoritos  |
